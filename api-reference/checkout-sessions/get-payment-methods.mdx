---
title: "Get Available Payment Methods"
openapi: "GET /v1/checkout/sessions/{checkout_session_id}/payment-methods"
---

Retrieve the payment methods available for a checkout session. Returns both the customer's enrolled (vaulted) payment methods and the generic payment methods enabled for the merchant's country and configuration.

## When to use

Call this endpoint after creating a checkout session to display available payment options to the customer. This is especially useful for custom (non-SDK) checkout UIs where you need to render payment method choices yourself.

## Authentication

<Note>
Requires both `public-api-key` and `private-secret-key` headers. See [Authentication](/api-reference/authentication) for details.
</Note>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `checkout_session_id` | string (path) | Yes | UUID of the checkout session |
| `category` | string (query) | No | Filter by category: `CARD`, `WALLET`, `BANK_TRANSFER` |

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Display name (e.g., `"VISA ****1111"` or `"Apple Pay"`) |
| `vaulted_token` | string/null | Token UUID if enrolled, `null` for generic methods |
| `type` | string | Method type: `CARD`, `APPLE_PAY`, `GOOGLE_PAY`, `NEQUI_ENROLLMENT`, `ACH`, etc. |
| `category` | string | Category: `CARD`, `WALLET`, `BANK_TRANSFER` |
| `icon` | string | URL to the payment method icon image |
| `checkout.sdk_required_action` | boolean | `false` = can process directly (vaulted card); `true` = needs SDK UI |
| `checkout.conditions.enabled` | boolean | Whether this method is enabled for the session |
| `card_data` | object/null | Card details (only present for `CARD` type with a `vaulted_token`) |
| `preferred` | boolean | Whether the customer has marked this method as preferred |

## Error responses

| Status | Code | Description | Resolution |
|--------|------|-------------|------------|
| 400 | `INVALID_REQUEST` | Invalid session ID | Verify the checkout session UUID |
| 401 | `INVALID_CREDENTIALS` | Invalid API keys | Verify your API key headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Confirm your account has Checkout API access |

## Notes

<Info>
Use the `sdk_required_action` field to determine your UI flow. When `false`, the payment can be processed directly with the vaulted token -- no SDK form is needed. When `true`, render the SDK checkout form.
</Info>

- Vaulted methods appear only when the checkout session was created with a `customer_id`.
- The `conditions.rules` array controls conditional visibility. `enabled: false` with rules means the method is conditionally disabled based on country, amount, or metadata filters.
- `card_data` is only present for enrolled CARD types; generic methods like "Card" or "Apple Pay" do not include it.
