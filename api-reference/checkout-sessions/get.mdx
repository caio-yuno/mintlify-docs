---
title: "Get Checkout Session"
openapi: "GET /v1/checkout/sessions/{checkout_session_id}"
---

Retrieve the details of an existing checkout session by its ID.

## When to use

Use this endpoint to check the status of a checkout session, verify whether it has been used, or retrieve its configuration after creation.

## Authentication

<Note>
Requires both `public-api-key` and `private-secret-key` headers. See [Authentication](/api-reference/authentication) for details.
</Note>

## Request example

<CodeGroup>
```bash cURL
curl -X GET https://api-sandbox.y.uno/v1/checkout/sessions/70bc6ef1-510b-452c-9fea-3894a58971ac \
  -H "public-api-key: YOUR_PUBLIC_API_KEY" \
  -H "private-secret-key: YOUR_PRIVATE_SECRET_KEY"
```

```javascript Node.js
const response = await fetch(
  "https://api-sandbox.y.uno/v1/checkout/sessions/70bc6ef1-510b-452c-9fea-3894a58971ac",
  {
    method: "GET",
    headers: {
      "public-api-key": "YOUR_PUBLIC_API_KEY",
      "private-secret-key": "YOUR_PRIVATE_SECRET_KEY",
    },
  }
);
const session = await response.json();
```

```python Python
import requests

response = requests.get(

        "private-secret-key": "YOUR_PRIVATE_SECRET_KEY",
    },

```
</CodeGroup>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `checkout_session_id` | string (path) | Yes | UUID of the checkout session to retrieve |

## Response

Returns the same structure as [Create Checkout Session](/api-reference/checkout-sessions/create).

```json
{
  "merchant_order_id": "order-001",
  "checkout_session": "70bc6ef1-510b-452c-9fea-3894a58971ac",
  "country": "CO",
  "payment_description": "Premium subscription",
  "customer_id": "5ac25323-9cc6-4247-bf04-df712b31e892",
  "callback_url": "https://www.example.com/callback?checkoutSession=70bc6ef1-510b-452c-9fea-3894a58971ac",
  "amount": {
    "currency": "COP",
    "value": 50000
  },
  "created_at": "2026-03-01T09:59:51.310340Z",
  "metadata": [
    { "key": "plan", "value": "premium" }
  ],
  "workflow": "SDK_CHECKOUT",
  "installments": {
    "plan_id": null,
    "plan": null
  },
  "used": false
}
```

| Field | Type | Description |
|-------|------|-------------|
| `checkout_session` | string | UUID of the session |
| `used` | boolean | `true` after a payment has been created from this session |
| `created_at` | string | ISO 8601 creation timestamp |

## Error responses

| Status | Code | Description | Resolution |
|--------|------|-------------|------------|
| 400 | `INVALID_REQUEST` | Invalid session ID format | Verify the `checkout_session_id` is a valid UUID |
| 401 | `INVALID_CREDENTIALS` | Invalid API keys | Verify `public-api-key` and `private-secret-key` headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Confirm your account has Checkout API access enabled |
| 404 | `NOT_FOUND` | Session does not exist | Check that the session ID is correct and belongs to your account |
