---
title: "Create Customer"
openapi: "POST /v1/customers"
---

Create a customer record in Yuno. The returned `id` is used to associate the customer with checkout sessions and enrolled payment methods.

## When to use

Create a customer before starting a payment flow when you need to vault payment methods, display saved cards in checkout, or track payment history per customer. The more customer data you provide, the better your approval rates and fraud prevention.

## Authentication

<Note>
Requires both `public-api-key` and `private-secret-key` headers. See [Authentication](/api-reference/authentication) for details.
</Note>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `merchant_customer_id` | string | Yes | Your external customer identifier (1-255 chars) |
| `email` | string | Yes | Customer email address (3-255 chars) |
| `first_name` | string | No | Customer first name (1-255 chars) |
| `last_name` | string | No | Customer last name (1-255 chars) |
| `gender` | string | No | `M`, `F`, or `NB` |
| `date_of_birth` | string | No | Format: `YYYY-MM-DD` |
| `country` | string | No | ISO 3166-1 alpha-2 country code |
| `nationality` | string | No | Nationality country code (ISO 3166-1 alpha-2) |
| `document` | object | No | Identity document with `document_type` and `document_number` |
| `phone` | object | No | Phone with `number` and `country_code` (calling code, e.g., `"57"`) |
| `billing_address` | object | No | Address with `address_line_1`, `state`, `city`, `zip_code` (all required if provided) |
| `shipping_address` | object | No | Same structure as `billing_address` |
| `metadata` | array | No | Key-value pairs for custom data |
| `merchant_customer_created_at` | string | No | ISO 8601 timestamp of when the customer was created in your system |

| Field | Type | Description |

| `updated_at` | string | ISO 8601 last update timestamp |

## Error responses

| Status | Code | Description | Resolution |
|--------|------|-------------|------------|
| 400 | `INVALID_REQUEST` | Missing or invalid fields | Check required fields (`merchant_customer_id`, `email`) and value constraints |
| 401 | `INVALID_CREDENTIALS` | Invalid API keys | Verify your API key headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Confirm your account has Customers API access |

## Notes

<Warning>
Creating a customer with a duplicate `merchant_customer_id` does **not** return an error -- it creates a new customer record with a different Yuno `id`. Use [Retrieve Customer by External ID](/api-reference/customers/get-by-external-id) to check for existing customers first.
</Warning>

- `document_type` values vary by country: `CC` (Colombia), `CPF` (Brazil), `SSN` (US), `DNI` (Argentina), `CURP` (Mexico).
- Providing complete customer data (address, document, phone) improves payment approval rates and fraud scoring.
