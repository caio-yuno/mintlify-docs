---
title: "Get Customer"
openapi: "GET /v1/customers/{customer_id}"
---

Retrieve a customer by their Yuno-generated UUID.

## When to use

Use this endpoint to fetch the full customer profile when you have the Yuno `id` (returned from [Create Customer](/api-reference/customers/create)). If you only have your external identifier, use [Retrieve Customer by External ID](/api-reference/customers/get-by-external-id) instead.

## Authentication

<Note>
Requires both `public-api-key` and `private-secret-key` headers. See [Authentication](/api-reference/authentication) for details.
</Note>

## Request example

<CodeGroup>
```bash cURL
curl -X GET https://api-sandbox.y.uno/v1/customers/5ac25323-9cc6-4247-bf04-df712b31e892 \
  -H "public-api-key: YOUR_PUBLIC_API_KEY" \
  -H "private-secret-key: YOUR_PRIVATE_SECRET_KEY"
```

```javascript Node.js
const response = await fetch(
  "https://api-sandbox.y.uno/v1/customers/5ac25323-9cc6-4247-bf04-df712b31e892",
  {
    method: "GET",
    headers: {
      "public-api-key": "YOUR_PUBLIC_API_KEY",
      "private-secret-key": "YOUR_PRIVATE_SECRET_KEY",
    },
  }
);
const customer = await response.json();
```

```python Python
import requests

response = requests.get(
    "https://api-sandbox.y.uno/v1/customers/5ac25323-9cc6-4247-bf04-df712b31e892",
    headers={
        "public-api-key": "YOUR_PUBLIC_API_KEY",
        "private-secret-key": "YOUR_PRIVATE_SECRET_KEY",
    },
)
customer = response.json()
```
</CodeGroup>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `customer_id` | string (path) | Yes | Yuno-generated customer UUID (36-64 chars) |

## Error responses

| Status | Code | Description | Resolution |
|--------|------|-------------|------------|
| 400 | `CUSTOMER_NOT_FOUND` | No customer matches the given ID | Verify the UUID is correct and was returned by Create Customer |
| 401 | `INVALID_CREDENTIALS` | Invalid API keys | Verify your API key headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Confirm your account has Customers API access |

<Warning>
Invalid customer IDs may return HTTP 400 with `CUSTOMER_NOT_FOUND` rather than 404. Always check both status codes.
</Warning>
