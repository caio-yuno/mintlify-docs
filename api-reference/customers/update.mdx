---
title: "Update Customer"
openapi: "PATCH /v1/customers/{customer_id}"
---

Update an existing customer's profile. Only the fields you include in the request body are modified; all other fields remain unchanged.

## When to use

Use this endpoint when a customer's information changes -- for example, updating their email, address, phone number, or identity document. Keeping customer data current improves payment approval rates and fraud scoring.

## Authentication

<Note>
Requires both `public-api-key` and `private-secret-key` headers. See [Authentication](/api-reference/authentication) for details.
</Note>

## Request example

<CodeGroup>
```bash cURL
curl -X PATCH https://api-sandbox.y.uno/v1/customers/5ac25323-9cc6-4247-bf04-df712b31e892 \
  -H "public-api-key: YOUR_PUBLIC_API_KEY" \
  -H "private-secret-key: YOUR_PRIVATE_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "Jonathan",
    "email": "jonathan.doe@example.com"
  }'
```

```javascript Node.js
const response = await fetch(
```


## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `customer_id` | string (path) | Yes | Yuno-generated customer UUID (36-64 chars) |
| `merchant_customer_id` | string | No | Update external merchant ID |
| `email` | string | No | Update email address |
| `first_name` | string | No | Update first name |
| `last_name` | string | No | Update last name |
| `gender` | string | No | Update gender (`M`, `F`, `NB`) |
| `date_of_birth` | string | No | Update date of birth (`YYYY-MM-DD`) |
| `country` | string | No | Update country code |
| `nationality` | string | No | Update nationality |
| `document` | object | No | Update identity document (both `document_type` and `document_number` required if provided) |
| `phone` | object | No | Update phone (both `number` and `country_code` required if provided) |
| `billing_address` | object | No | Update billing address (`address_line_1`, `state`, `city`, `zip_code` required if provided) |
| `shipping_address` | object | No | Update shipping address |
| `metadata` | array | No | Replace metadata (full array replacement, not merge) |

## Response

Returns the full updated customer object.

```json
{
  "id": "5ac25323-9cc6-4247-bf04-df712b31e892",
  "merchant_customer_id": "cust-12345",
  "first_name": "Jonathan",
  "last_name": "Doe",
  "email": "jonathan.doe@example.com",
  "country": "CO",
  "created_at": "2026-03-01T09:59:35.135690Z",
  "updated_at": "2026-03-01T09:59:45.538586Z"
}
```

## Error responses

| Status | Code | Description | Resolution |
|--------|------|-------------|------------|
| 400 | `INVALID_REQUEST` | Invalid request body or field validation errors | Check field constraints and formats |
| 400 | `CUSTOMER_NOT_FOUND` | Customer does not exist | Verify the customer UUID |
| 401 | `INVALID_CREDENTIALS` | Invalid API keys | Verify your API key headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Confirm your account has Customers API access |

## Notes

<Warning>
Updating `metadata` replaces the **entire** metadata array. To preserve existing metadata keys, include them in the update request alongside any new keys.
</Warning>

- Nested objects (`document`, `phone`, `billing_address`) require all of their internal required fields if included.
- The `updated_at` timestamp changes; `created_at` remains the same.
- You cannot update the Yuno-generated `id`.
