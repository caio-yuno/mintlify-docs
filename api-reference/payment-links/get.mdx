---
title: "Retrieve Payment Link"
openapi: "GET /v1/payment-links/{payment_link_id}"
---

Retrieves the details of an existing payment link by its unique code.

## When to use

Use this endpoint to check the current status of a payment link, verify if a customer has completed payment, or retrieve the `checkout_url` to re-share with a customer.

## Authentication

All requests require the following headers:

| Header | Description |
|--------|-------------|
| `public-api-key` | Your Yuno public API key |
| `private-secret-key` | Your Yuno private secret key |

## Request example

<CodeGroup>
```bash cURL
curl -X GET https://api-sandbox.y.uno/v1/payment-links/e1512b4d-9057-4c73-8e49-921c51eb5ba2 \
  -H "public-api-key: your-public-key" \
  -H "private-secret-key: your-secret-key"
```

```javascript Node.js
const response = await fetch(
  'https://api-sandbox.y.uno/v1/payment-links/e1512b4d-9057-4c73-8e49-921c51eb5ba2',
  {
    headers: {
      'public-api-key': 'your-public-key',
      'private-secret-key': 'your-secret-key'
    }
  }
);
const paymentLink = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api-sandbox.y.uno/v1/payment-links/e1512b4d-9057-4c73-8e49-921c51eb5ba2',
    headers={
        'public-api-key': 'your-public-key',
        'private-secret-key': 'your-secret-key'
    }
)
payment_link = response.json()
```
</CodeGroup>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `payment_link_id` | string (path) | Yes | The payment link UUID (the `code` returned when the link was created) |

## Response

Returns the full payment link object including current status and payment details:

```json
{
  "code": "e1512b4d-9057-4c73-8e49-921c51eb5ba2",
  "country": "CO",
  "status": "CREATED",
  "amount": { "currency": "COP", "value": 50000 },
  "description": "Invoice #1234",
  "merchant_order_id": "order-1234",
  "capture": true,
  "one_time_use": true,
  "payment_method_types": ["CARD"],
  "checkout_url": "https://checkout.sandbox.y.uno/payment?session=2f6d9754-43d0-4dca-b023-6aba4b107179",
  "callback_url": "https://yoursite.com/payment/complete",
  "payments": null,
  "payments_number": 0,
  "organization_code": "org-uuid-here",
  "account_code": "493e9374-510a-4201-9e09-de669d75f256"
}
```

When the link has been used, the `payments` array contains payment IDs and `payments_number` reflects the count.

## Error responses

| Status | Description | Resolution |
|--------|-------------|------------|
| 400 | Invalid request format | Ensure the payment link code is a valid UUID |
| 401 | Invalid or missing API keys | Verify `public-api-key` and `private-secret-key` headers |
| 404 | Payment link not found | Confirm the payment link code exists in your account |

<Note>
The retrieve response includes an `organization_code` field not present in the create response. Use the `status` field to determine the link's current state: `CREATED` (active and unused), `PAID` (payment completed), `CANCELED` (manually canceled), or `EXPIRED` (past `finish_at` date).
</Note>
