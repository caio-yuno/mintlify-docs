---
title: "Retrieve Payment Method"
openapi: "GET /v1/customers/{customer_id}/payment-methods/{payment_method_id}"
---

Retrieve the full details of a single enrolled payment method by its vaulted token.

## When to use

Use this endpoint to get complete information about a specific payment method, including its enrollment details, provider information, and card fingerprint. This endpoint returns more detail than the [List](/api-reference/payment-methods/list) endpoint.

## Authentication

<Note>
Requires both `public-api-key` and `private-secret-key` headers. See [Authentication](/api-reference/authentication) for details.
</Note>

## Request example

<CodeGroup>
```bash cURL
curl -X GET https://api-sandbox.y.uno/v1/customers/5ac25323-9cc6-4247-bf04-df712b31e892/payment-methods/4945a392-de69-4484-a224-8427343d0a1e \
  -H "public-api-key: YOUR_PUBLIC_API_KEY" \
  -H "private-secret-key: YOUR_PRIVATE_SECRET_KEY"
```

```javascript Node.js
const response = await fetch(
  "https://api-sandbox.y.uno/v1/customers/5ac25323-9cc6-4247-bf04-df712b31e892/payment-methods/4945a392-de69-4484-a224-8427343d0a1e",
  {
    method: "GET",
    headers: {
      "public-api-key": "YOUR_PUBLIC_API_KEY",
      "private-secret-key": "YOUR_PRIVATE_SECRET_KEY",

  }

```

```python Python

        "public-api-key": "YOUR_PUBLIC_API_KEY",

```
</CodeGroup>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `customer_id` | string (path) | Yes | Yuno customer UUID (36-64 chars) |
| `payment_method_id` | string (path) | Yes | The `vaulted_token` from enrollment (36-64 chars) |

| Field | Type | Description |
|-------|------|-------------|
| `vaulted_token` | string | The payment method's unique identifier |
| `status` | string | `ENROLLED` or `UNENROLLED` |
| `sub_status` | string | Detailed status (populated in single retrieve, `null` in list) |
| `card_data.issuer` | string | Issuing bank name. **Note:** this field is named `issuer` in single retrieve but `issuer_name` in the list endpoint |
| `card_data.fingerprint` | string | Card fingerprint UUID (populated here, `null` in list endpoint) |
| `card_data.country_code` | string | Card issuing country code (populated here, `null` in list endpoint) |

<Info>
This endpoint returns richer data than [List Enrolled Payment Methods](/api-reference/payment-methods/list). Key differences: `fingerprint`, `sub_status`, `country_code`, and `idempotency_key` are populated here but `null` in list responses. The issuer field is named `issuer` here vs `issuer_name` in list.
</Info>

## Error responses

| Status | Code | Description | Resolution |
|--------|------|-------------|------------|
| 400 | `INVALID_REQUEST` | Invalid request format | Verify the UUID format for both path parameters |
| 400 | `CUSTOMER_NOT_FOUND` | Customer does not exist | Verify the customer UUID |
| 401 | `INVALID_CREDENTIALS` | Invalid API keys | Verify your API key headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Confirm your account has Payment Methods API access |
