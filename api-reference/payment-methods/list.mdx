---
title: "List Enrolled Payment Methods"
openapi: "GET /v1/customers/{customer_id}/payment-methods"
---

List all enrolled (vaulted) payment methods for a customer. Only methods with `ENROLLED` status are returned; unenrolled methods are excluded.

## When to use

Use this endpoint to display a customer's saved payment methods in your UI, or to check which cards are available before initiating a payment. For richer details on a specific method (fingerprint, sub_status), use [Retrieve Payment Method](/api-reference/payment-methods/get) instead.

## Authentication

<Note>
Requires both `public-api-key` and `private-secret-key` headers. See [Authentication](/api-reference/authentication) for details.
</Note>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `customer_id` | string (path) | Yes | Yuno customer UUID (36-64 chars) |

| Field | Type | Description |
|-------|------|-------------|
| `payment_methods` | array | Array of enrolled payment method objects |
| `icon` | string | URL to the payment method icon (present in list, absent in single retrieve) |
| `card_data.issuer_name` | string | Issuing bank name. **Note:** named `issuer_name` here vs `issuer` in single retrieve |
| `card_data.fingerprint` | null | Always `null` in list response. Use [Retrieve Payment Method](/api-reference/payment-methods/get) for fingerprint |

<Warning>
The response field naming differs between this endpoint and single retrieve. The issuer field is `issuer_name` in list responses but `issuer` in [Retrieve Payment Method](/api-reference/payment-methods/get). Account for both field names in your integration code.
</Warning>

### List vs Single retrieve differences

| Field | List (this endpoint) | Single Retrieve |
|-------|---------------------|-----------------|
| `icon` | Present | Absent |
| Issuer field name | `issuer_name` | `issuer` |
| `fingerprint` | `null` | Populated |
| `sub_status` | `null` | Populated |
| `country_code` in card_data | `null` | Populated |
| `idempotency_key` | `null` | Populated |
| Response wrapper | `{ payment_methods: [...] }` | Direct object |

## Error responses

| Status | Code | Description | Resolution |
|--------|------|-------------|------------|
| 400 | `INVALID_REQUEST` | Invalid request | Verify the customer UUID format |
| 401 | `INVALID_CREDENTIALS` | Invalid API keys | Verify your API key headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Confirm your account has Payment Methods API access |

## Notes

- Only `ENROLLED` methods are returned. Unenrolled methods are excluded from the list.
- The response is wrapped in `{ payment_methods: [...] }`, not a bare array.
- For card fingerprint and full issuer details, call [Retrieve Payment Method](/api-reference/payment-methods/get) on individual items.
