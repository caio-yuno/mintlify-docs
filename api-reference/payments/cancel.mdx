---
title: "Cancel Payment"
openapi: "POST /v1/payments/{payment_id}/cancel"
---

Cancel a payment that has `PENDING` status with an `AUTHORIZED` sub-status. This releases the held funds back to the customer.

## When to use

Use this endpoint to void an authorization that you no longer intend to capture. Common scenarios include order cancellations, inventory issues, or when the customer requests cancellation before fulfillment.

## Authentication

| Header | Type | Required | Description |
|---|---|---|---|
| `public-api-key` | string | Yes | Your public API key from the Yuno dashboard |
| `private-secret-key` | string | Yes | Your private secret key from the Yuno dashboard |
| `X-Idempotency-Key` | string | Yes | UUID v4 to prevent duplicate processing |

## Key parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `payment_id` | string (UUID v4) | Yes | Path parameter. The payment to cancel |
| `merchant_reference` | string | Yes | Your reference for this cancellation (3-255 characters) |
| `reason` | string | No | One of: `DUPLICATE`, `FRAUDULENT`, `REQUESTED_BY_CUSTOMER` |
| `description` | string | No | Cancellation description (3-255 characters) |
| `response_additional_data.receipt` | boolean | No | Whether to generate a receipt |
| `response_additional_data.receipt_language` | string | No | Receipt language: `ES`, `EN`, or `PT` |

## Response

A successful request returns a `200` status code with a `CANCEL` transaction. The payment status updates to `CANCELED`.

<Warning>
Only payments with `PENDING` status and `AUTHORIZED` sub-status can be canceled. Payments pending due to 3DS authentication **cannot** be canceled -- they will auto-resolve to either verified or canceled. For completed payments, use [Refund](/api-reference/payments/refund) instead.
</Warning>

## Error responses

| Status | Code | Description | Resolution |
|---|---|---|---|
| 400 | `INVALID_REQUEST` | Payment is not in a cancelable state | Verify the payment has `PENDING`/`AUTHORIZED` status. Use [Refund](/api-reference/payments/refund) for `SUCCEEDED` payments |
| 400 | `INVALID_PARAMETERS` | Invalid UUID format or parameter values | Ensure IDs are valid UUID v4 format |
| 401 | `INVALID_CREDENTIALS` | Missing or invalid API keys | Verify your API key headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Contact Yuno support |
