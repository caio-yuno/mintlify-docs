Create a payment after a checkout session has been established. Supports single-step (authorize and capture) and two-step (authorize only) flows.

## When to use

Use this endpoint to process a payment after your customer has selected a payment method through a checkout session. Choose single-step (`capture: true`) for immediate charges or two-step (`capture: false`) when you need to authorize first and capture later (for example, after order fulfillment).

## Authentication

<Note>All payment endpoints require both your public and private API keys.</Note>

| Header | Type | Required | Description |
|---|---|---|---|
| `public-api-key` | string | Yes | Your public API key from the Yuno dashboard |
| `private-secret-key` | string | Yes | Your private secret key from the Yuno dashboard |
| `X-Idempotency-Key` | string | Yes | UUID v4 to prevent duplicate payment processing |

## Key parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `description` | string | Yes | Payment description (3-255 characters) |
| `country` | string | Yes | Customer's country in ISO 3166-1 alpha-2 format. Supported: `AR`, `BO`, `BR`, `CL`, `CO`, `CR`, `EC`, `SV`, `GT`, `HN`, `MX`, `NI`, `PA`, `PY`, `PE`, `US`, `UY` |
| `merchant_order_id` | string | Yes | Your unique order identifier (3-255 characters) |
| `amount.currency` | string | Yes | ISO 4217 currency code. Supported: `ARS`, `BOV`, `BOB`, `BRL`, `CLP`, `COP`, `CRC`, `USD`, `SVC`, `GTQ`, `HNL`, `MXN`, `NIO`, `PAB`, `PYG`, `PEN`, `UYU` |
| `amount.value` | number | Yes | Amount in smallest currency unit (for example, `10000` = $100.00 USD) |
| `workflow` | string | Yes | One of `SDK_CHECKOUT`, `DIRECT`, or `REDIRECT` |
| `payment_method.type` | string | Yes | Payment method type (for example, `CARD`) |
| `payment_method.detail.card.capture` | boolean | No | `true` for single-step (default), `false` for two-step auth-then-capture |
| `payment_method.detail.card.card_data` | object | Yes | Card details: `number`, `expiration_month`, `expiration_year`, `security_code`, `holder_name` |
| `customer_payer` | object | No | Customer information including `first_name`, `last_name`, `email`, `document`, `phone`, `billing_address` |
| `metadata` | array | No | Array of `{key, value}` objects for custom data |
| `callback_url` | string | No | URL for payment callbacks |

A successful request returns a `201` status code with the full payment object.

<Warning>
The Create Payment response returns `transactions` as a **single object**. The [Get Payment](/api-reference/payments/get) endpoint returns `transactions` as an **array**. Handle both structures in your integration.
</Warning>

## Error responses

| Status | Code | Description | Resolution |
|---|---|---|---|
| 400 | `INVALID_REQUEST` | Malformed request body or missing required fields | Verify all required fields are present and correctly formatted |
| 400 | `INVALID_PARAMETERS` | Invalid parameter value (for example, unsupported country code) | Check parameter values against supported options |
| 401 | `INVALID_CREDENTIALS` | Missing or invalid API keys | Verify your `public-api-key` and `private-secret-key` headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized for this API | Contact Yuno support to verify your account permissions |

If you use `metadata` for routing logic, you must also set the same metadata in the Checkout Session. Setting it only in the Payment request will not activate routing rules.
</Note>

<Warning>
Amount values are in the **smallest currency unit**. For USD, `10000` represents $100.00. For BRL, `5000` represents R$50.00.
</Warning>
