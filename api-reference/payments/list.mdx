---
title: "List Payments"
openapi: "GET /v1/payments"
---

Retrieve a list of payments with optional filters. Use query parameters to narrow results by date, status, or other criteria.

## When to use

Use this endpoint to list and filter payments for reporting, reconciliation, or building admin dashboards. Combine with pagination parameters for large result sets.

## Authentication

| Header | Type | Required | Description |
|---|---|---|---|
| `public-api-key` | string | Yes | Your public API key from the Yuno dashboard |
| `private-secret-key` | string | Yes | Your private secret key from the Yuno dashboard |

## Request example

<CodeGroup>

```bash cURL
curl -X GET "https://api-sandbox.y.uno/v1/payments" \
  -H "public-api-key: YOUR_PUBLIC_API_KEY" \
  -H "private-secret-key: YOUR_PRIVATE_SECRET_KEY"
```

```javascript Node.js
const response = await fetch("https://api-sandbox.y.uno/v1/payments", {
  method: "GET",
  headers: {
    "public-api-key": "YOUR_PUBLIC_API_KEY",
    "private-secret-key": "YOUR_PRIVATE_SECRET_KEY",
  },
});

const payments = await response.json();
```

```python Python
import requests

response = requests.get(
    "https://api-sandbox.y.uno/v1/payments",
    headers={
        "public-api-key": "YOUR_PUBLIC_API_KEY",
        "private-secret-key": "YOUR_PRIVATE_SECRET_KEY",
    },
)

payments = response.json()
```

</CodeGroup>

## Error responses

| Status | Code | Description | Resolution |
|---|---|---|---|
| 401 | `INVALID_CREDENTIALS` | Missing or invalid API keys | Verify your `public-api-key` and `private-secret-key` headers |
| 403 | `AUTHORIZATION_REQUIRED` | Merchant not authorized | Contact Yuno support to verify account permissions |
