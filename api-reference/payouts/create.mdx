---
title: "Create Payout"
openapi: "POST /v1/payouts"
---

Creates a new payout to send funds to a previously registered recipient.

## When to use

Use this endpoint to disburse funds to sellers, vendors, or other recipients in your marketplace or platform. Payouts are processed through the recipient's registered payment method and country-specific rails.

## Authentication

All requests require the following headers:

| Header | Description |
|--------|-------------|
| `public-api-key` | Your Yuno public API key |
| `private-secret-key` | Your Yuno private secret key |

## Request example

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.y.uno/v1/payouts \
  -H "public-api-key: your-public-key" \
  -H "private-secret-key: your-secret-key" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": {
      "currency": "BRL",
      "value": 500.00
    },
    "country": "BR",
    "recipient_id": "d1e2f3a4-b5c6-7d8e-9f0a-1b2c3d4e5f6a",
    "description": "Seller payout - Order #1234",
    "merchant_reference": "payout-ref-001",
    "purpose": "SELLER_PAYMENT"
  }'
```

```javascript Node.js
const response = await fetch('https://api-sandbox.y.uno/v1/payouts', {

});
const payout = await response.json();
```

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `amount.currency` | string | Yes | ISO 4217 currency code (e.g., `BRL`, `COP`, `MXN`) |
| `amount.value` | number | Yes | Payout amount |
| `country` | string | Yes | ISO 3166-1 alpha-2 country code of the recipient |
| `recipient_id` | string | Yes | UUID of the registered recipient to send funds to |
| `description` | string | No | Payout description for your records |
| `merchant_reference` | string | No | Your internal payout reference for reconciliation |
| `purpose` | string | No | Purpose of the payout (e.g., `SELLER_PAYMENT`) |

## Response

A successful request returns `201 Created` with the payout object in `PENDING` status:

```json
{
  "id": "e2f3a4b5-c6d7-8e9f-0a1b-2c3d4e5f6a7b",
  "status": "PENDING",
  "amount": {
    "currency": "BRL",
    "value": 500.00
  },
  "country": "BR",
  "recipient_id": "d1e2f3a4-b5c6-7d8e-9f0a-1b2c3d4e5f6a",
  "merchant_reference": "payout-ref-001",
  "created_at": "2026-03-01T16:00:00.000Z"
}
```

## Payout status lifecycle

| Status | Description |
|--------|-------------|
| `PENDING` | Payout created and awaiting processing |
| `SUCCEEDED` | Funds successfully delivered to the recipient |
| `FAILED` | Payout processing failed |
| `CANCELLED` | Payout was canceled before processing |

## Error responses

| Status | Description | Resolution |
|--------|-------------|------------|
| 400 | Invalid request body or missing required fields | Verify amount, country, and recipient_id are valid |
| 401 | Invalid or missing API keys | Verify `public-api-key` and `private-secret-key` headers |
