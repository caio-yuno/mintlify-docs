---
title: "Get Payout"
openapi: "GET /v1/payouts/{payout_id}"
---

Retrieves the details and current status of a payout by its unique identifier.

## When to use

Use this endpoint to check whether a payout has been successfully processed, is still pending, or has failed. This is essential for building payout tracking dashboards and reconciliation workflows.

## Authentication

All requests require the following headers:

| Header | Description |
|--------|-------------|
| `public-api-key` | Your Yuno public API key |
| `private-secret-key` | Your Yuno private secret key |

## Request example

<CodeGroup>
```bash cURL
curl -X GET https://api-sandbox.y.uno/v1/payouts/e2f3a4b5-c6d7-8e9f-0a1b-2c3d4e5f6a7b \
  -H "public-api-key: your-public-key" \
  -H "private-secret-key: your-secret-key"
```

```javascript Node.js
const response = await fetch(
  'https://api-sandbox.y.uno/v1/payouts/e2f3a4b5-c6d7-8e9f-0a1b-2c3d4e5f6a7b',
  {
    headers: {
      'public-api-key': 'your-public-key',
      'private-secret-key': 'your-secret-key'
    }
  }
);
const payout = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api-sandbox.y.uno/v1/payouts/e2f3a4b5-c6d7-8e9f-0a1b-2c3d4e5f6a7b',
    headers={
        'public-api-key': 'your-public-key',
        'private-secret-key': 'your-secret-key'
    }
)
payout = response.json()
```
</CodeGroup>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `payout_id` | string (path) | Yes | The unique UUID of the payout |

## Response

Returns the full payout object with current status:

```json
{
  "id": "e2f3a4b5-c6d7-8e9f-0a1b-2c3d4e5f6a7b",
  "status": "SUCCEEDED",
  "amount": {
    "currency": "BRL",
    "value": 500.00
  },
  "country": "BR",
  "recipient_id": "d1e2f3a4-b5c6-7d8e-9f0a-1b2c3d4e5f6a",
  "merchant_reference": "payout-ref-001",
  "created_at": "2026-03-01T16:00:00.000Z"
}
```

## Error responses

| Status | Description | Resolution |
|--------|-------------|------------|
| 401 | Invalid or missing API keys | Verify `public-api-key` and `private-secret-key` headers |
| 404 | Payout not found | Confirm the payout ID is a valid UUID and exists in your account |
