---
title: "Retrieve Subscription"
openapi: "GET /v1/subscriptions/{subscription_id}"
---

Retrieves the full details of an existing subscription by its unique identifier.

## When to use

Use this endpoint to check the current status of a subscription, view billing cycle progress, or verify payment method details. This is useful for building subscription management dashboards or syncing subscription state with your system.

## Authentication

All requests require the following headers:

| Header | Description |
|--------|-------------|
| `public-api-key` | Your Yuno public API key |
| `private-secret-key` | Your Yuno private secret key |

## Request example

<CodeGroup>
```bash cURL
curl -X GET https://api-sandbox.y.uno/v1/subscriptions/0c7fed3e-ee0d-4d34-9547-778be4ec0798 \
  -H "public-api-key: your-public-key" \
  -H "private-secret-key: your-secret-key"
```

```javascript Node.js
const response = await fetch(
  'https://api-sandbox.y.uno/v1/subscriptions/0c7fed3e-ee0d-4d34-9547-778be4ec0798',
  {
    headers: {
      'public-api-key': 'your-public-key',
      'private-secret-key': 'your-secret-key'
    }
  }
);
const subscription = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api-sandbox.y.uno/v1/subscriptions/0c7fed3e-ee0d-4d34-9547-778be4ec0798',
    headers={
        'public-api-key': 'your-public-key',
        'private-secret-key': 'your-secret-key'
    }
)
subscription = response.json()
```
</CodeGroup>

## Key parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `subscription_id` | string (path) | Yes | The unique UUID of the subscription to retrieve |

## Error responses

| Status | Description | Resolution |
|--------|-------------|------------|
| 400 | Invalid request format | Ensure the subscription ID is a valid UUID |
| 401 | Invalid or missing API keys | Verify `public-api-key` and `private-secret-key` headers |
| 404 | Subscription not found | Confirm the subscription ID exists in your account |

<Note>
In the retrieve response, `payment_method` is returned as an **array** containing full card details (token, vaulted_token, card_data), unlike the create response which returns a single object. The `payments` field contains an array of associated payment IDs generated by completed billing cycles.
</Note>
