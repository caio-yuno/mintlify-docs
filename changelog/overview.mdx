## Versioning Policy

Yuno follows semantic versioning (SemVer) for all SDKs:

| Version Part | Meaning | Example |
|-------------|---------|---------|
| **Major** (X.0.0) | Breaking changes requiring migration | 2.0.0 -> 3.0.0 |
| **Minor** (0.X.0) | New features, backward compatible | 2.4.0 -> 2.5.0 |
| **Patch** (0.0.X) | Bug fixes, no new features | 2.5.0 -> 2.5.1 |

<Note>
The REST API is versioned via the URL path (`/v1/`). Breaking changes result in a new API version. Non-breaking additions (new fields, endpoints) are added to the current version.
</Note>

## Deprecation Policy

- **SDKs:** Deprecated versions are supported for 6 months after the next major release
- **API endpoints:** Deprecated endpoints remain functional for 12 months with deprecation warnings in response headers
- **Dashboard features:** Deprecated features show in-app migration notices

<Warning>
Subscribe to the Yuno changelog RSS feed or enable email notifications in **Dashboard > Settings > Notifications** to receive automatic updates about new releases and deprecations.
</Warning>
