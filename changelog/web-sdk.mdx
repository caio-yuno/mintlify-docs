## Overview

Release history for the Yuno Web SDK (`@yuno-payments/sdk-web`). Install or update via npm:

```bash
npm install @yuno-payments/sdk-web@latest
```

---

## v2.5.0 (February 2026)

### New Features
- **Secure Fields styling API v2:** Full CSS property support including `::placeholder`, `:focus`, and `:hover` pseudo-selectors
- **Field-level validation events:** `onFieldChange` now includes `isValid`, `isEmpty`, and `cardBrand` properties
- **Auto-detect card brand:** Card number field emits brand detection as the user types

### Improvements
- Reduced SDK bundle size by 18% (tree-shaking improvements)
- Improved 3DS challenge iframe rendering on mobile Safari

### Bug Fixes
- Fixed `mountLiteCheckout` failing silently when `elementSelector` contains special characters
- Resolved memory leak when calling `unmount()` and re-mounting multiple times

---

## v2.4.0 (January 2026)

### New Features
- **Dark mode support:** Pass `theme.mode: 'dark'` to automatically adjust all UI elements
- **Language detection:** SDK auto-detects browser locale and sets language (overridable)
- **Enrollment widget:** `mountEnrollment()` method for card vaulting flows

### Improvements
- 3DS challenge popup now respects `prefers-color-scheme` media query
- Improved accessibility: ARIA labels, keyboard navigation, screen reader support

### Bug Fixes
- Fixed race condition when `continuePayment()` is called before SDK initialization completes
- Resolved CORS preflight issue with certain CDN configurations

## v2.2.0 (November 2025)

### New Features
- **Installment selector:** Full Checkout and Lite Checkout display installment options for supported countries
- **Payment method filtering:** Pass `allowedPaymentMethods` to limit displayed methods

```javascript
yuno.mountLiteCheckout({
  elementSelector: '#form',
  allowedPaymentMethods: ['CARD', 'PIX'],
});
```

### Improvements
- Improved loading performance: SDK initialization is 40% faster
- Added TypeScript generics for callback types

## v2.0.0 (September 2025)

### Breaking Changes

<Warning>
v2.0.0 introduces breaking changes to the initialization API. See the migration guide below.
</Warning>

- **New initialization API:** `Yuno.initialize()` now accepts an object argument and returns a Promise
- **Removed:** `startCheckout()` and `mountCheckout()` methods (replaced by `mountFullCheckout()` and `mountLiteCheckout()`)
- **Removed:** String-based `Yuno.initialize('key')` pattern

### Migration from v1.x

```javascript
// v1.x (removed)
const yuno = Yuno.initialize('public-key');
yuno.startCheckout({ checkoutSession: 'id', elementSelector: '#form' });
yuno.mountCheckout();

// v2.x (current)
const yuno = await Yuno.initialize({
  publicApiKey: 'public-key',
  checkoutSession: 'id',
  country: 'BR',
});
yuno.mountLiteCheckout({ elementSelector: '#form' });
```

### New Features
- Async initialization with proper error handling
- Built-in TypeScript support
- Automatic checkout session validation on init
