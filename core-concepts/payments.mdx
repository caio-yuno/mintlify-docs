## Overview

A payment represents a charge against a customer's selected payment method. Payments are always created within the context of a [checkout session](/core-concepts/checkout-sessions) and are processed through the provider determined by your [routing configuration](/core-concepts/providers).

## Creating a Payment

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.y.uno/v1/payments \
  -H "public-api-key: $YUNO_PUBLIC_KEY" \
  -H "private-secret-key: $YUNO_PRIVATE_KEY" \
  -H "X-Idempotency-Key: pay-order-9876" \
  -H "Content-Type: application/json" \
  -d '{
    "checkout_session": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "payment_method": {
      "type": "PIX"
    },
    "amount": {
      "currency": "BRL",
      "value": 100.00
    },
    "country": "BR",
    "customer": {
      "document": {
        "document_type": "CPF",
        "document_number": "12345678901"
      },
      "email": "customer@example.com"
    }
  }'
```

## Payment Statuses

| Status | Meaning | Next Actions |
|--------|---------|-------------|
| `PENDING` | Awaiting customer action or provider confirmation | Wait for [webhook](/core-concepts/webhooks) update |
| `AUTHORIZED` | Funds reserved (auth-only mode with `capture: false`) | [Capture or cancel](/guides/direct-api/capture-and-cancel) |
| `SUCCEEDED` | Funds captured successfully | [Refund](/guides/direct-api/refunds) if needed |
| `DECLINED` | Provider or issuer declined the charge | Retry with different method or provider |
| `REJECTED` | Yuno validation or fraud rules blocked the payment | Fix request and retry |
| `CANCELLED` | Cancelled before completion | Create new payment |
| `REFUNDED` | Captured funds fully returned to customer | No further action |
| `PARTIALLY_REFUNDED` | Part of the captured funds returned | Additional refunds possible |

## Auto-Capture vs Manual Capture

By default, payments are auto-captured. Set `"capture": false` to use manual capture (auth-only), which reserves funds without charging.

```json
{
  "checkout_session": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "capture": false,
  "payment_method": { "type": "CARD" },
  "amount": { "currency": "BRL", "value": 200.00 },
  "country": "BR"
}
```

### Capturing an Authorized Payment

```bash
curl -X POST https://api-sandbox.y.uno/v1/payments/{payment_id}/transactions/{transaction_id}/capture \
  -H "public-api-key: $YUNO_PUBLIC_KEY" \
  -H "private-secret-key: $YUNO_PRIVATE_KEY" \
  -H "X-Idempotency-Key: capture-order-9876" \
  -H "Content-Type: application/json" \
  -d '{
    "merchant_reference": "capture-9876"
  }'
```

<Warning>
Capture authorized payments within the provider's hold period (typically 5-7 days for cards). Uncaptured authorizations are voided automatically.
</Warning>

## Refunds

Refund a succeeded payment partially or fully:

```bash
curl -X POST https://api-sandbox.y.uno/v1/payments/{payment_id}/transactions/{transaction_id}/refund \
  -H "public-api-key: $YUNO_PUBLIC_KEY" \
  -H "private-secret-key: $YUNO_PRIVATE_KEY" \
  -H "X-Idempotency-Key: refund-order-9876" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": { "currency": "BRL", "value": 50.00 },
    "reason": "REQUESTED_BY_CUSTOMER",
    "merchant_reference": "refund-9876"
  }'
```

<Note>
Refund availability depends on the payment method and provider. Some methods (e.g., Boleto, OXXO) may not support refunds through the API.
</Note>

## Cancellations

Cancel a payment that has not yet been captured:

```bash
curl -X POST https://api-sandbox.y.uno/v1/payments/{payment_id}/transactions/{transaction_id}/cancel \
  -H "public-api-key: $YUNO_PUBLIC_KEY" \
  -H "private-secret-key: $YUNO_PRIVATE_KEY" \
  -H "X-Idempotency-Key: cancel-order-9876" \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "REQUESTED_BY_CUSTOMER",
    "merchant_reference": "cancel-9876"
  }'
```
