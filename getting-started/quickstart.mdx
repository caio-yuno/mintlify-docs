## Prerequisites

- A Yuno account with sandbox access
- Your API keys from the [Yuno Dashboard](https://dashboard.y.uno)

## Create Your First Payment

<Steps>
  <Step title="Get your API keys">
    Log in to the [Yuno Dashboard](https://dashboard.y.uno) and navigate to **Settings > API Keys**. You need three values:

    - `public-api-key` — Used for client-side SDK initialization and server-side calls
    - `private-secret-key` — Used for server-side API calls only
    - `account-code` — Your account identifier

    <Warning>
      Never expose your `private-secret-key` in client-side code or version control.
    </Warning>
  </Step>

  <Step title="Create a checkout session">
    Create a checkout session on your server to initialize the payment flow.

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api-sandbox.y.uno/v1/checkout/sessions \
      -H "public-api-key: your-public-api-key" \
      -H "private-secret-key: your-private-secret-key" \
      -H "account-code: your-account-code" \
      -H "Content-Type: application/json" \
      -d '{
        "amount": {
          "currency": "BRL",
          "value": 100.00
        },
        "country": "BR",
        "merchant_order_id": "order-001",
        "payment_description": "Test payment"
      }'
    ```

    ```javascript Node.js
    const response = await fetch('https://api-sandbox.y.uno/v1/checkout/sessions', {
      method: 'POST',
      headers: {
        'public-api-key': 'your-public-api-key',
        'private-secret-key': 'your-private-secret-key',
        'account-code': 'your-account-code',
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        amount: { currency: 'BRL', value: 100.00 },
        country: 'BR',
        merchant_order_id: 'order-001',
        payment_description: 'Test payment',
      }),
    });

    const session = await response.json();
    // Use session.checkout_session for SDK initialization
    ```
    </CodeGroup>

    <Note>
      This endpoint returns HTTP **200** (not 201). The checkout session ID is returned in the response body.
    </Note>
  </Step>

  <Step title="Create a payment">
    Use the checkout session to create a payment.

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api-sandbox.y.uno/v1/payments \
      -H "public-api-key: your-public-api-key" \
      -H "private-secret-key: your-private-secret-key" \
      -H "account-code: your-account-code" \
      -H "Content-Type: application/json" \
      -d '{
        "checkout_session": "your-checkout-session-id",
        "payment_method": {
          "type": "CARD"
        },
        "amount": {
          "currency": "BRL",
          "value": 100.00
        },
        "country": "BR",
        "customer": {
          "document": {
            "document_type": "CPF",
            "document_number": "12345678901"
          },
          "email": "customer@example.com"
        }
      }'
    ```
    </CodeGroup>
  </Step>

  <Step title="Handle the response">
    Check the payment status in the response:

    - `SUCCEEDED` — Payment completed successfully
    - `PENDING` — Payment is awaiting confirmation (common for PIX, Boleto)
    - `DECLINED` — Payment was declined by the provider
    - `REJECTED` — Payment was rejected due to validation errors

    For asynchronous payment methods (PIX, Boleto, OXXO), set up [webhooks](/guides/webhooks/overview) to receive status updates.
  </Step>
</Steps>
