## Overview

Refunds return funds to the customer after a payment has been captured. Yuno supports both full refunds (entire amount) and partial refunds (a portion of the original amount).

<Note>
Refunds can only be issued for payments with status `SUCCEEDED`. For authorized but uncaptured payments, use [Cancel](/guides/direct-api/capture-and-cancel) instead -- it releases the hold without processing fees.
</Note>

## Endpoint

```
POST https://api-sandbox.y.uno/v1/payments/{payment_id}/transactions/{transaction_id}/refund
```

### Required Headers

| Header | Value |
|--------|-------|
| `public-api-key` | Your public API key |
| `private-secret-key` | Your private secret key |
| `X-Idempotency-Key` | Unique key to prevent duplicate refunds |
| `Content-Type` | `application/json` |

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `amount` | object | No | Amount to refund. Omit for full refund. |
| `amount.currency` | string | Yes (if amount) | ISO 4217 currency code |
| `amount.value` | number | Yes (if amount) | Amount to refund |
| `description` | string | No | Description of the refund (3-255 chars) |
| `reason` | string | No | Reason code: `DUPLICATE`, `FRAUDULENT`, or `REQUESTED_BY_CUSTOMER` |
| `merchant_reference` | string | Yes | Your reference for this refund (3-255 chars) |

## Create a Full Refund

Omit the `amount` field to refund the full captured amount.

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.y.uno/v1/payments/{payment_id}/transactions/{transaction_id}/refund \
  -H "public-api-key: $YUNO_PUBLIC_KEY" \
  -H "private-secret-key: $YUNO_PRIVATE_KEY" \
  -H "X-Idempotency-Key: refund-order-123-full" \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "REQUESTED_BY_CUSTOMER",
    "description": "Customer requested full refund",
    "merchant_reference": "refund-order-123"
  }'
```

```javascript Node.js
const refund = await fetch(
  `https://api-sandbox.y.uno/v1/payments/${paymentId}/transactions/${transactionId}/refund`,
  {
    method: 'POST',
    headers: {
      'public-api-key': process.env.YUNO_PUBLIC_KEY,
      'private-secret-key': process.env.YUNO_PRIVATE_KEY,
      'X-Idempotency-Key': 'refund-order-123-full',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      reason: 'REQUESTED_BY_CUSTOMER',
      description: 'Customer requested full refund',
      merchant_reference: 'refund-order-123',
    }),
  }
);

const refundResult = await refund.json();
// refundResult.type === 'REFUND'
// refundResult.status === 'SUCCEEDED'
```

```python Python
import requests
import os

refund = requests.post(
    f'https://api-sandbox.y.uno/v1/payments/{payment_id}/transactions/{transaction_id}/refund',
    headers={
        'public-api-key': os.environ['YUNO_PUBLIC_KEY'],
        'private-secret-key': os.environ['YUNO_PRIVATE_KEY'],
        'X-Idempotency-Key': 'refund-order-123-full',
        'Content-Type': 'application/json',
    },
    json={
        'reason': 'REQUESTED_BY_CUSTOMER',
        'description': 'Customer requested full refund',
        'merchant_reference': 'refund-order-123',
    },
)

refund_result = refund.json()
```
</CodeGroup>

## Create a Partial Refund

Specify the `amount` object to refund a portion of the payment.

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.y.uno/v1/payments/{payment_id}/transactions/{transaction_id}/refund \
  -H "public-api-key: $YUNO_PUBLIC_KEY" \
  -H "private-secret-key: $YUNO_PRIVATE_KEY" \
  -H "X-Idempotency-Key: refund-order-123-partial" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": { "currency": "USD", "value": 25.00 },
    "reason": "REQUESTED_BY_CUSTOMER",
    "description": "Partial refund - item returned",
    "merchant_reference": "refund-order-123-item-a"
  }'
```

```javascript Node.js
const partialRefund = await fetch(
  `https://api-sandbox.y.uno/v1/payments/${paymentId}/transactions/${transactionId}/refund`,
  {
    method: 'POST',
    headers: {
      'public-api-key': process.env.YUNO_PUBLIC_KEY,
      'private-secret-key': process.env.YUNO_PRIVATE_KEY,
      'X-Idempotency-Key': 'refund-order-123-partial',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      amount: { currency: 'USD', value: 25.00 },
      reason: 'REQUESTED_BY_CUSTOMER',
      description: 'Partial refund - item returned',
      merchant_reference: 'refund-order-123-item-a',
    }),
  }
);
```

```python Python
partial_refund = requests.post(
    f'https://api-sandbox.y.uno/v1/payments/{payment_id}/transactions/{transaction_id}/refund',
    headers={
        'public-api-key': os.environ['YUNO_PUBLIC_KEY'],
        'private-secret-key': os.environ['YUNO_PRIVATE_KEY'],
        'X-Idempotency-Key': 'refund-order-123-partial',
        'Content-Type': 'application/json',
    },
    json={
        'amount': {'currency': 'USD', 'value': 25.00},
        'reason': 'REQUESTED_BY_CUSTOMER',
        'description': 'Partial refund - item returned',
        'merchant_reference': 'refund-order-123-item-a',
    },
)
```
</CodeGroup>

## Refund Statuses

| Status | Description |
|--------|-------------|
| `SUCCEEDED` | Refund processed and funds returned |
| `PENDING` | Refund submitted, awaiting provider confirmation |
| `DECLINED` | Refund could not be processed |

## Multiple Partial Refunds

You can issue multiple partial refunds against a single payment, as long as the total refunded amount does not exceed the original captured amount:

```
Original payment:       $100.00
First partial refund:   - $30.00  (remaining: $70.00)
Second partial refund:  - $20.00  (remaining: $50.00)
Third partial refund:   - $50.00  (remaining: $0.00, fully refunded)
```

<Warning>
Attempting to refund more than the remaining capturable amount will result in an error. Track cumulative refunds on your side for accurate reconciliation. Do not execute a refund while another is in progress -- wait for the current operation to complete.
</Warning>

## Refund Timelines

Refund processing time depends on the payment method and provider:

| Payment Method | Typical Timeline |
|---------------|------------------|
| Credit Card | 5-10 business days |
| Debit Card | 3-5 business days |
| PIX | Instant to 1 business day |
| Bank Transfer | 3-7 business days |

## Error Handling

| HTTP Status | Error Code | Cause | Resolution |
|-------------|-----------|-------|------------|
| `400` | `INVALID_REQUEST` | Missing required fields | Check `merchant_reference` is provided |
| `400` | `PAYMENT_NOT_REFUNDABLE` | Payment not in succeeded state | Verify payment status before refunding |
| `400` | `REFUND_AMOUNT_EXCEEDED` | Refund amount > remaining capturable amount | Reduce the refund amount |
| `401` | `INVALID_CREDENTIALS` | Invalid API keys | Check authentication headers |
| `409` | `CONFLICT` | Duplicate idempotency key | Use a new idempotency key |

## Best Practices

- **Always include a descriptive `reason`** for audit and reconciliation purposes
- **Use webhooks** to track refund status changes asynchronously
- **Implement idempotency keys** to prevent duplicate refund requests
- **Prefer cancel/void over refund** for same-day reversals to minimize fees
- **Keep a log mapping refund IDs** to order/item-level details
- **Do not overlap refund operations** -- wait for one to complete before starting another
