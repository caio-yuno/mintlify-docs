## YunoConfig

Configuration object passed to `Yuno.initialize()` in your `Application` class.

```kotlin
Yuno.initialize(
    this,
    "your-public-api-key",
    config = YunoConfig(
        saveCardEnabled = false,
        language = YunoLanguage.ENGLISH,
        styles = null
    )
)
```

### Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `application` | `Application` | Yes | -- | Your Application instance |
| `publicApiKey` | `String` | Yes | -- | Your Yuno public API key |
| `config` | `YunoConfig` | Yes | -- | Configuration data class |

### YunoConfig properties

```kotlin
data class YunoConfig(
    val saveCardEnabled: Boolean = false,
    val language: YunoLanguage? = null,
    val styles: YunoStyles? = null
)
```

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `saveCardEnabled` | `Boolean` | `false` | Show "Save card for future payments" checkbox |
| `language` | `YunoLanguage?` | Device locale | UI language: `ENGLISH`, `SPANISH`, `PORTUGUESE`, `INDONESIAN`, `MALAY`, `THAI` |
| `styles` | `YunoStyles?` | Yuno defaults | Visual customization. See [YunoAppearance](#yunoappearance-customization) |

<Note>
Starting from SDK version 2.11.0, `cardFlow` is no longer part of `YunoConfig`. Card flow configuration is handled exclusively through the CheckoutBuilder.
</Note>

<Warning>
Never embed your private secret key in the Android app. Use only the public API key for client-side initialization.
</Warning>

## Checkout methods

### startCheckout

Initializes the payment flow. Call inside `onCreate()` of the Activity that manages the checkout.

```kotlin
startCheckout(
    checkoutSession = "checkout_session_id",
    countryCode = "CO",
    callbackPaymentState = { paymentState, paymentSubState ->
        handlePaymentState(paymentState, paymentSubState)
    },
    merchantSessionId = null
)
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `checkoutSession` | `String` | Yes | Checkout session ID |
| `countryCode` | `String` | Yes | ISO 3166-1 alpha-2 country code |
| `callbackPaymentState` | `(String?, String?) -> Unit` | No | Callback for payment state/sub-state updates |
| `merchantSessionId` | `String?` | No | Merchant tracking identifier |

### startPayment

Starts the payment process after the user selects a method.

```kotlin
startPayment(
    showStatusYuno = true,
    callbackOTT = { ott -> createPaymentOnServer(ott) },
    callBackTokenWithInformation = { tokenModel -> /* detailed token info */ }
)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `showStatusYuno` | `Boolean` | Show payment status screen in SDK |
| `callbackOTT` | `(String?) -> Unit` | Callback with one-time token (OTT) |
| `callBackTokenWithInformation` | `(OneTimeTokenModel?) -> Unit` | Callback with detailed token info |

### continuePayment

Continue the payment flow after creating the payment server-side. Required when `sdk_action_required` is `true`.

```kotlin
continuePayment(
    showPaymentStatus = true,
    checkoutSession = null,
    countryCode = null,
    callbackPaymentState = { paymentState, paymentSubState ->
        handlePaymentState(paymentState, paymentSubState)
    }
)
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `showPaymentStatus` | `Boolean` | `true` | Show the SDK payment status screen |
| `checkoutSession` | `String?` | `null` | Override checkout session (optional) |
| `countryCode` | `String?` | `null` | Override country code (optional) |
| `callbackPaymentState` | `(String?, String?) -> Unit` | `null` | Callback for payment state updates |

## Jetpack Compose components

### PaymentMethodListViewComponent

Displays available payment methods in Full Checkout.

```kotlin
@Composable
fun PaymentMethodListViewComponent(
    activity: Activity,
    modifier: Modifier? = null,
    onPaymentSelected: (Boolean) -> Unit,
    onUnEnrollSuccess: (Boolean) -> Unit = {}
)
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `activity` | `Activity` | Yes | The hosting Activity |
| `modifier` | `Modifier?` | No | Layout and appearance customization |
| `onPaymentSelected` | `(Boolean) -> Unit` | Yes | `true` when a method is selected (enable pay button) |
| `onUnEnrollSuccess` | `(Boolean) -> Unit` | No | Called when a stored method is successfully removed |

```kotlin
var paymentMethodIsSelected by remember { mutableStateOf(false) }

Column(
    modifier = Modifier
        .weight(1f)
        .verticalScroll(rememberScrollState())
) {
    PaymentMethodListViewComponent(
        activity = activity,
        onPaymentSelected = { isSelected ->
            paymentMethodIsSelected = isSelected
        },
        onUnEnrollSuccess = { success ->
            if (success) {
                coroutineScope.launch {
                    snackbarHostState.showSnackbar("Payment method removed")
                }
            }
        },
    )
}
```

<Warning>
Always wrap `PaymentMethodListViewComponent` in a `Column` with `.verticalScroll(rememberScrollState())`. Without this, the list may not render or scroll properly.
</Warning>

## Render mode integration

The render mode provides advanced UI flexibility, allowing developers to control how the payment form appears.

### startPaymentRender

```kotlin
class PaymentActivity : Activity() {

    private lateinit var fragmentContainer: FrameLayout

    fun setupPaymentRender(checkoutSession: String) {
        Yuno.startPaymentRender(
            activity = this,
            checkoutSession = checkoutSession,
            countryCode = "CO",
            listener = paymentRenderListener
        )
    }

    private val paymentRenderListener = object : YunoPaymentRenderListener {
        override fun onPaymentMethodView(view: View) {
            fragmentContainer.addView(view)
        }

        override fun onPaymentStateChange(state: String, subState: String?) {
            handlePaymentState(state, subState)
        }
    }
}
```

## Callback interfaces

### YunoSeamlessListener

Implement this for Seamless Checkout payment creation callbacks.

```kotlin
class CheckoutActivity : AppCompatActivity(), YunoSeamlessListener {

    override fun yunoCreatePayment(
        oneTimeToken: String,
        tokenWithInformation: YunoTokenData
    ) {
        PaymentApi.createPayment(oneTimeToken) { success ->
            if (success) {
                Yuno.continuePayment()
            }
        }
    }

    override fun yunoPaymentResult(result: PaymentResult) {
        when (result.status) {
            PaymentStatus.SUCCEEDED -> navigateToConfirmation(result.paymentId)
            PaymentStatus.FAILED -> showError(result.errorMessage)
            PaymentStatus.PROCESSING -> showProcessingState()
            PaymentStatus.CANCELLED -> showCancelledState()
        }
    }
}
```

### OnEnrollmentListener

Implement this to receive enrollment lifecycle events.

```kotlin
class EnrollmentActivity : AppCompatActivity(), OnEnrollmentListener {

    override fun onEnrollmentStateChange(state: EnrollmentState) {
        when (state) {
            EnrollmentState.ENROLLED -> showSuccess("Card saved")
            EnrollmentState.ENROLL_FAILED -> showError("Could not save card")
            EnrollmentState.DECLINED -> showError("Card declined")
            EnrollmentState.REJECTED -> showError("Card rejected")
            EnrollmentState.EXPIRED -> showError("Session expired")
            EnrollmentState.CANCELLED -> dismiss()
        }
    }
}
```

## Enrollment flows

### Starting enrollment

```kotlin
private val enrollmentLauncher = registerForActivityResult(
    EnrollmentLauncher()
) { result: EnrollmentResult ->
    handleEnrollmentResult(result)
}

fun startEnrollment(sessionId: String) {
    enrollmentLauncher.launch(
        EnrollmentParams(
            enrollmentSession = sessionId,
            countryCode = "CO",
            customerId = "customer-001"
        )
    )
}
```

### Enrollment statuses

| Status | Description |
|--------|-------------|
| `CREATED` | Enrollment session created |
| `READY_TO_ENROLL` | Customer has submitted details, ready to enroll |
| `ENROLLED` | Payment method successfully enrolled and vaulted |
| `ENROLL_FAILED` | Enrollment attempt failed |
| `EXPIRED` | Session expired before completion |
| `REJECTED` | Enrollment rejected by provider |
| `DECLINED` | Enrollment declined |
| `UNENROLLED` | Previously enrolled method has been removed |

## Activity result handling

Register launchers in your Activity or Fragment before `onCreate()` completes.

```kotlin
class CheckoutActivity : AppCompatActivity() {

    private val fullCheckoutLauncher = registerForActivityResult(
        FullCheckoutLauncher()
    ) { result -> handlePaymentResult(result) }

    private val seamlessLauncher = registerForActivityResult(
        SeamlessCheckoutLauncher()
    ) { result -> handlePaymentResult(result) }

    private val enrollmentLauncher = registerForActivityResult(
        EnrollmentLauncher()
    ) { result -> handleEnrollmentResult(result) }
}
```

<Warning>
In Fragments, register launchers in `onCreate()`, not `onViewCreated()`. Registering after the Fragment is `STARTED` causes an `IllegalStateException`.
</Warning>

## Payment states

```kotlin
const val PAYMENT_STATE_SUCCEEDED = "SUCCEEDED"
const val PAYMENT_STATE_FAIL = "FAIL"
const val PAYMENT_STATE_PROCESSING = "PROCESSING"
const val PAYMENT_STATE_REJECT = "REJECT"
const val PAYMENT_STATE_INTERNAL_ERROR = "INTERNAL_ERROR"
const val PAYMENT_STATE_STATE_CANCELED_BY_USER = "CANCELED"
```

| State | Description |
|-------|-------------|
| `SUCCEEDED` | Payment completed successfully |
| `FAIL` | Payment failed due to an error |
| `PROCESSING` | Async payment in progress |
| `REJECT` | Payment rejected |
| `INTERNAL_ERROR` | Unexpected server-side error |
| `CANCELED` | User cancelled the payment flow |

### Sync payment methods (Google Pay)

For synchronous payment methods like Google Pay, when a user cancels:

- **SDK Status**: Returns `CANCELED`
- **Backend status**: Remains `PENDING` until PSP timeout

### Async payment methods (PIX)

For asynchronous methods like PIX, when a user closes the QR window:

- **SDK Status**: Returns `PROCESSING` with sub-status `CLOSED_BY_USER`
- **Backend status**: Remains `PENDING`, QR code stays valid until expiry

## YunoAppearance customization

```kotlin
val appearance = YunoAppearance(
    primaryColor = Color.parseColor("#6200EE"),
    backgroundColor = Color.WHITE,
    textColor = Color.BLACK,
    errorColor = Color.parseColor("#E53E3E"),
    borderColor = Color.parseColor("#E0E0E0"),
    cornerRadiusDp = 12,
    elevation = 4f,
    fontFamily = "sans-serif-medium",
    fontSizeSp = 14,
    buttonStyle = ButtonStyle.ROUNDED
)
```

### Color properties

| Property | Type | Description |
|----------|------|-------------|
| `primaryColor` | `Int` (Color) | Primary accent color for buttons, links, and active states |
| `backgroundColor` | `Int` (Color) | Background color of the checkout sheet |
| `textColor` | `Int` (Color) | Primary text color |
| `errorColor` | `Int` (Color) | Color for error messages and invalid field states |
| `borderColor` | `Int` (Color) | Border color for input fields |

### Typography properties

| Property | Type | Description |
|----------|------|-------------|
| `fontFamily` | `String` | Android font family name |
| `fontSizeSp` | `Int` | Base font size in sp |

### Layout properties

| Property | Type | Description |
|----------|------|-------------|
| `cornerRadiusDp` | `Int` | Corner radius in dp |
| `elevation` | `Float` | Elevation in dp for the checkout sheet |
| `buttonStyle` | `ButtonStyle` | `ROUNDED` or `RECTANGULAR` |

### Dark mode

The SDK respects the system dark mode setting by default. To force a specific mode:

```kotlin
val appearance = YunoAppearance(
    forceDarkMode = true,
    primaryColor = Color.parseColor("#BB86FC"),
    backgroundColor = Color.parseColor("#121212"),
    textColor = Color.WHITE
)
```

|----------|-------------|
| `"CARD"` | Credit and debit cards |

## Error codes

| Code | Description | Resolution |
|------|-------------|------------|
| `NETWORK_ERROR` | Network connectivity issue | Check device connectivity and retry |
| `AUTHENTICATION_FAILED` | Invalid public API key | Verify key in Dashboard > API Keys |
| `SESSION_EXPIRED` | Checkout session has expired | Create a new checkout session |
| `INVALID_CONFIGURATION` | SDK misconfiguration | Check initialization parameters |
| `PROVIDER_ERROR` | Payment provider rejected | Review error details and payment method |
| `UNKNOWN_ERROR` | Unexpected error | Enable logging and check Logcat output |

```
# Yuno SDK

# Google Pay (required if using Google Pay)
-keep class com.google.android.gms.wallet.** { *; }
```

<Note>
The SDK ships with consumer ProGuard rules. These additional rules are only needed if you encounter obfuscation issues.
</Note>
