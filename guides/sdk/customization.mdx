## Overview

Yuno SDK provides extensive customization options so the checkout experience matches your brand. You can configure themes, colors, typography, and layout through SDK initialization, inline CSS injection, or the no-code [Checkout Builder](/platform/dashboard/checkout-builder) in the Dashboard.

There are two places where customization applies:

- **Iframe content** -- the card form rendered inside Yuno's secure iframe. Customize via the `card.styles` parameter or `options.styles` in Secure Fields
- **External elements** -- components outside the iframe (payment method list, buttons, status screens). Customize by injecting CSS into your page

## Theme Configuration

Pass a `theme` object during SDK initialization to customize global appearance:

```javascript
const yuno = await Yuno.initialize({
  publicApiKey: 'your-public-api-key',
  countryCode: 'CO',
  language: 'en',
  theme: {
    primary: '#0066FF',
    secondary: '#1A1A2E',
    background: '#FFFFFF',
    text: '#333333',
    error: '#E53E3E',
    success: '#38A169',
    borderRadius: '8px',
    fontFamily: 'Inter, system-ui, sans-serif',
  },
});
```

## Available Theme Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `primary` | `string` | `#0066FF` | Primary action color (buttons, links, active states) |
| `secondary` | `string` | `#6B7280` | Secondary elements color |
| `background` | `string` | `#FFFFFF` | Form background color |
| `text` | `string` | `#1F2937` | Primary text color |
| `error` | `string` | `#EF4444` | Error state color for fields and messages |
| `success` | `string` | `#10B981` | Success state color |
| `borderRadius` | `string` | `4px` | Border radius for inputs and buttons |
| `fontFamily` | `string` | `'Inter', 'DM Sans', sans-serif` | Font family for all text elements |
| `fontSize` | `string` | `14px` | Base font size |
| `mode` | `"light"` \| `"dark"` | `"light"` | Color mode. See [Dark Mode](#dark-mode) |

## Dark Mode

Enable dark mode by setting the `mode` property:

```javascript
theme: {
  mode: 'dark',
  primary: '#60A5FA',
  background: '#1F2937',
  text: '#F9FAFB',
}
```

<Note>
When `mode: 'dark'` is set, the SDK automatically adjusts contrast ratios for accessibility compliance. You can still override individual colors. Dark mode also responds to the system `prefers-color-scheme` media query.
</Note>

## Input Design Types

Configure how field labels and placeholders behave. This applies to all checkout types except Secure Fields (which uses `designType` in field creation):

```javascript
// For Full, Seamless, and Lite Checkout
yuno.startCheckout({
  inputDesignType: 'float-label',
  // ... other config
});
```

For Secure Fields, set the `designType` when creating each field:

```javascript
const cardNumber = secureFields.create({
  name: 'pan',
  designType: 'float-label',
  options: { /* ... */ },
});
```

| Design Type | Behavior |
|-------------|----------|
| `float-label` | Label floats above the field when focused or filled; appears as placeholder when empty |
| `float-label-static` | Label is always displayed above the field |
| `label-placeholder` | Label displays only as placeholder text inside the field |

## Card Form Styling (Iframe)

Inject custom CSS into the card form iframe via the `card.styles` property:

```javascript
yuno.startCheckout({
  // ... other config
  card: {
    type: 'extends',
    styles: `
      #root {
        .Yuno-fieldset__box {
          .Yuno-input__content {
            background: #f9fafb;
            border: 1px solid #d1d5db;
            border-radius: 8px;
            padding: 12px;
            font-size: 16px;
          }
        }
        .Yuno-button.Yuno-modal-bottom__button {
          background: #0066FF;
          border-radius: 8px;
          font-weight: 600;
        }
      }
    `,
  },
});
```

### CSS Variables

Yuno uses CSS custom properties that you can override:

```css
:root {
  --yuno-purple: #3E4FE0;
  --yuno-purple-80: #4763ea;
  --yuno-purple-50: #9ab2f6;
  --yuno-purple-30: #c3d0fa;
  --yuno-purple-10: #dde4fc;
  --yuno-off-white: #f0f0f0;
  --yuno-black-text: #313131;
  --yuno-card-background: #F6F7FB;
  --Neutral-W: #FFFFFF;
  --Neutral-B: #282A30;
  --Grey-2: #ECEFF2;
}
```

## Secure Fields Styling

For Secure Fields, use the `options.styles` parameter when creating each field:

```javascript
const cardNumber = secureFields.create({
  name: 'pan',
  options: {
    placeholder: '0000 0000 0000 0000',
    styles: `
      #root {
        .Yuno-fieldset__box {
          .Yuno-input__content {
            background: white;
            border: 1px solid #d1d5db;
            border-radius: 8px;
            padding: 12px;
            font-family: 'Inter', sans-serif;
            font-size: 16px;
          }
        }
      }
    `,
  },
});
```

You can also style Secure Fields using the `styles` object on the `mountSecureFields` method if using the older API:

```javascript
const secureFields = yuno.mountSecureFields({
  styles: {
    base: {
      fontSize: '16px',
      color: '#333',
      fontFamily: 'Inter, sans-serif',
      padding: '12px',
      '::placeholder': { color: '#9CA3AF' },
    },
    focus: {
      borderColor: '#0066FF',
      boxShadow: '0 0 0 2px rgba(0, 102, 255, 0.2)',
    },
    error: {
      color: '#EF4444',
      borderColor: '#EF4444',
    },
    valid: {
      borderColor: '#10B981',
    },
  },
});
```

### Secure Fields Style States

| State | Description |
|-------|-------------|
| `base` | Default field appearance |
| `focus` | When the field is focused |
| `error` | When the field has a validation error |
| `valid` | When the field passes validation |
| `::placeholder` | Placeholder text styling (nested inside `base`) |

## Layout Options

Control the checkout layout for Full Checkout and Lite Checkout:

```javascript
yuno.mountFullCheckout({
  elementSelector: '#yuno-checkout',
  layout: {
    direction: 'vertical',
    showHeader: true,
    showPaymentMethodIcons: true,
    compactMode: false,
  },
});
```

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `direction` | `"vertical"` \| `"horizontal"` | `"vertical"` | Layout direction for payment methods |
| `showHeader` | `boolean` | `true` | Show the checkout header |
| `showPaymentMethodIcons` | `boolean` | `true` | Show payment method brand icons |
| `compactMode` | `boolean` | `false` | Use compact layout for smaller containers |

## Localization

The SDK automatically detects the user's browser locale. Override it explicitly:

```javascript
const yuno = await Yuno.initialize({
  publicApiKey: 'your-public-api-key',
  countryCode: 'BR',
  language: 'pt',
});
```

Supported languages:

| Code | Language |
|------|----------|
| `en` | English |
| `es` | Spanish |
| `pt` | Portuguese |

## Complete Customization Example

```javascript
const yuno = await Yuno.initialize({
  publicApiKey: 'your-public-api-key',
  countryCode: 'CO',
  language: 'es',
  theme: {
    mode: 'light',
    primary: '#6200EE',
    secondary: '#03DAC6',
    background: '#FFFFFF',
    text: '#1F2937',
    error: '#B00020',
    success: '#00C853',
    borderRadius: '12px',
    fontFamily: '"Poppins", system-ui, sans-serif',
    fontSize: '15px',
  },
});

yuno.startSeamlessCheckout({
  checkoutSession: 'session-uuid',
  elementSelector: '#checkout',
  countryCode: 'CO',
  language: 'es',
  showLoading: true,
  showPaymentStatus: true,
  inputDesignType: 'float-label',

  renderMode: {
    type: 'element',
    elementSelector: '#checkout-form',
  },

  card: {
    type: 'extends',
    cardSaveEnable: true,
    hideCardholderName: false,
    styles: `
      #root {
        .Yuno-fieldset__box .Yuno-input__content {
          background: #fafafa;
          border: 2px solid #e0e0e0;
          border-radius: 12px;
          padding: 14px;
          font-family: 'Poppins', sans-serif;
          transition: border-color 0.2s ease;
        }
        .Yuno-fieldset__box .Yuno-input__content:focus-within {
          border-color: #6200EE;
        }
        .Yuno-button.Yuno-modal-bottom__button {
          background: #6200EE;
          border-radius: 12px;
          font-family: 'Poppins', sans-serif;
          font-weight: 600;
          padding: 16px;
        }
      }
    `,
    texts: {
      cardNumberLabel: 'Numero de tarjeta',
      expiryLabel: 'Fecha de vencimiento',
      cvvLabel: 'Codigo de seguridad',
    },
  },

  async yunoCreatePayment(oneTimeToken) {
    // ... payment creation
    yuno.continuePayment({ showPaymentStatus: true });
  },

  yunoPaymentResult(data) {
    console.log('Result:', data.status);
  },
});
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Custom styles are not applying">
    - Ensure your CSS string is passed to `card.styles` (for iframe content)
    - For external elements, inject CSS directly into your page
    - Use `#root` as the top-level selector in iframe CSS
    - Check that `!important` is used if existing styles have high specificity
  </Accordion>

  <Accordion title="Theme colors do not match expectations">
    - Verify the `theme` object is passed during `Yuno.initialize()`, not later
    - Check that SDK-level `card.styles` is not overriding theme properties
    - If using Checkout Builder, remember that programmatic themes take precedence
  </Accordion>

  <Accordion title="Dark mode is not working">
    - Ensure `mode: 'dark'` is set in the `theme` object
    - Override `background` and `text` colors explicitly for best results
    - Test with the browser's `prefers-color-scheme: dark` media query
  </Accordion>
</AccordionGroup>
