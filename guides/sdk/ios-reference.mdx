## YunoConfig

Configuration class passed to `Yuno.initialize()` during app startup.

```swift
import YunoSDK

Yuno.initialize(
    apiKey: "your-public-api-key",
    config: YunoConfig(
        cardFormType: .oneStep,
        appearance: appearance,
        saveCardEnabled: false,
        keepLoader: false
    ),
    callback: { (success: Bool) in }
)
```

### Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `apiKey` | `String` | Yes | -- | Your Yuno public API key |
| `config` | `YunoConfig` | Yes | -- | Configuration object |
| `callback` | `(Bool) -> Void` | Yes | -- | Called when initialization completes |

### YunoConfig properties

```swift
final class YunoConfig {
    let cardFormType: CardFormType
    let appearance: Yuno.Appearance
    let saveCardEnabled: Bool
    let keepLoader: Bool
}
```

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `cardFormType` | `CardFormType` | `.oneStep` | Card entry flow: `.oneStep` (single form) or `.multiStep` (step-by-step) |
| `appearance` | `Yuno.Appearance` | Yuno defaults | Visual customization. See [YunoAppearance](#yunotheme-customization) |
| `saveCardEnabled` | `Bool` | `false` | Show a "Save card for future payments" checkbox |
| `keepLoader` | `Bool` | `false` | Keep SDK loader visible while merchant creates payment |

<Warning>
Never include your private secret key in the iOS app. Use only the public API key for client-side initialization.
</Warning>

<Note>
If your app uses a `UISceneDelegate`, place `Yuno.initialize()` in your `SceneDelegate`, not `AppDelegate`.
</Note>

## Delegate protocols

### YunoPaymentDelegate

Base protocol for receiving payment lifecycle events. Implement this in your view controller.

```swift
protocol YunoPaymentDelegate: AnyObject {
    var checkoutSession: String { get }
    var countryCode: String { get }
    var language: String? { get }
    var viewController: UIViewController? { get }
    func yunoCreatePayment(with token: String)
    func yunoCreatePayment(with token: String, information: [String: Any])
    func yunoPaymentResult(_ result: Yuno.Result)
}
```

| Property / Method | Type | Description |
|-------------------|------|-------------|
| `checkoutSession` | `String` | Current checkout session ID |
| `countryCode` | `String` | ISO 3166-1 alpha-2 country code |
| `language` | `String?` | Language code: `"en"`, `"es"`, `"pt"`, `"id"`, `"ms"`, `"th"` |
| `viewController` | `UIViewController?` | The presenting view controller |
| `yunoCreatePayment(with:)` | Method | Called with the one-time token (OTT) for server-side payment creation |
| `yunoCreatePayment(with:information:)` | Method | Called with OTT and additional payment method metadata |
| `yunoPaymentResult(_:)` | Method | Called with the final payment result |

### YunoPaymentFullDelegate

Extends `YunoPaymentDelegate` for Full Checkout. Implement this to receive payment method selection events.

```swift
protocol YunoPaymentFullDelegate: YunoPaymentDelegate {
    func yunoDidSelect(paymentMethod: YunoSDK.PaymentMethodSelected)
    func yunoDidUnenrollSuccessfully(_ success: Bool)
    func yunoUpdatePaymentMethodsViewHeight(_ height: CGFloat)
}
```

| Method | Parameters | Description |
|--------|------------|-------------|
| `yunoDidSelect(paymentMethod:)` | `PaymentMethodSelected` | Called when the user selects a payment method |
| `yunoDidUnenrollSuccessfully(_:)` | `Bool` | Called when a stored payment method is successfully removed |
| `yunoUpdatePaymentMethodsViewHeight(_:)` | `CGFloat` | Called when `getPaymentMethodViewAsync()` is invoked and whenever the view height changes |

### YunoSeamlessDelegate

Implement this for Seamless Checkout to handle token generation and payment results.

```swift
extension CheckoutViewController: YunoSeamlessDelegate {

    func yunoCreatePayment(
        oneTimeToken: String,
        tokenWithInformation: YunoTokenData
    ) {
        PaymentAPI.createPayment(token: oneTimeToken) { success in
            if success {
                Yuno.continuePayment()
            }
        }
    }

    func yunoPaymentResult(_ result: YunoPaymentResult) {
        handlePaymentResult(result)
    }

    func yunoDidFail(error: YunoError) {
        showError(message: error.localizedDescription)
    }
}
```

| Method | Parameters | Description |
|--------|------------|-------------|
| `yunoCreatePayment(oneTimeToken:tokenWithInformation:)` | `String`, `YunoTokenData` | Called when a token is generated. Create payment server-side, then call `Yuno.continuePayment()` |
| `yunoPaymentResult(_:)` | `YunoPaymentResult` | Called with the final payment outcome |
| `yunoDidFail(error:)` | `YunoError` | Called when the SDK encounters an error |

### YunoEnrollmentDelegate

Implement this to receive enrollment results.

```swift
extension EnrollmentViewController: YunoEnrollmentDelegate {

    func yunoDidCompleteEnrollment(result: YunoEnrollmentResult) {
        switch result.status {
        case .enrolled:
            let vaultedToken = result.vaultedToken
            saveVaultedToken(vaultedToken)
            showSuccess("Card saved successfully")
        case .enrollFailed:
            showError("Could not save card")
        case .declined:
            showError("Card was declined")
        case .rejected:
            showError("Enrollment rejected")
        case .expired:
            refreshSessionAndRetry()
        @unknown default:
            break
        }
    }

    func yunoDidFailEnrollment(error: YunoError) {
        showError(error.localizedDescription)
    }
}
```

## SDK methods

### getPaymentMethodViewAsync

Returns the payment methods view for Full Checkout. Automatically returns `UIView` (UIKit) or `some View` (SwiftUI).

```swift
let view = await Yuno.getPaymentMethodViewAsync(delegate: self)
```

### startPayment

Starts the payment process after the user selects a payment method.

```swift
Yuno.startPayment(showPaymentStatus: true)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `showPaymentStatus` | `Bool` | Show the SDK payment status screen after completion |

### continuePayment

Continue the payment flow after creating the payment server-side. Required when `sdk_action_required` is `true`.

```swift
Yuno.continuePayment()
```

### receiveDeeplink

Handle deep links for payment methods that redirect to external apps.

```swift
func application(_ app: UIApplication,
                 open url: URL,
                 options: [UIApplication.OpenURLOptionsKey : Any] = [:]) -> Bool {
    guard url.scheme == "yoururlscheme" else { return false }
    return Yuno.receiveDeeplink(url)
}
```

## Checkout configuration

### Full Checkout

```swift
Yuno.startFullCheckout(
    from: self,
    checkoutSession: "session-uuid",
    countryCode: "CO",
    applePay: YunoApplePayConfig(
        merchantId: "merchant.com.yourapp.pay",
        countryCode: "CO"
    ),
    delegate: self
)
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | `UIViewController` | Yes | Presenting view controller |
| `checkoutSession` | `String` | Yes | Checkout session ID from your server |
| `countryCode` | `String` | Yes | ISO 3166-1 alpha-2 country code |
| `applePay` | `YunoApplePayConfig` | No | Apple Pay configuration |
| `delegate` | `YunoPaymentDelegate` | Yes | Delegate to receive payment results |

### Seamless Checkout

```swift
Yuno.startSeamlessCheckout(
    from: self,
    checkoutSession: "session-uuid",
    paymentMethodType: "CARD",
    countryCode: "CO",
    vaultedToken: nil,
    delegate: self
)
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | `UIViewController` | Yes | Presenting view controller |
| `checkoutSession` | `String` | Yes | Checkout session ID |
| `paymentMethodType` | `String` | Yes | Payment method type (e.g., `"CARD"`, `"PIX"`) |
| `countryCode` | `String` | Yes | ISO 3166-1 alpha-2 country code |
| `vaultedToken` | `String?` | No | Vaulted token for saved payment methods |
| `delegate` | `YunoSeamlessDelegate` | Yes | Delegate to receive payment lifecycle events |

## Enrollment flows

### Starting enrollment

```swift
Yuno.startEnrollment(
    from: self,
    enrollmentSession: "enrollment-session-uuid",
    countryCode: "CO",
    customerId: "customer-001",
    delegate: self
)
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | `UIViewController` | Yes | Presenting view controller |
| `enrollmentSession` | `String` | Yes | Enrollment session ID from your server |
| `countryCode` | `String` | Yes | ISO 3166-1 alpha-2 country code |
| `customerId` | `String` | Yes | Your customer identifier |
| `delegate` | `YunoEnrollmentDelegate` | Yes | Delegate to receive enrollment results |

### Enrollment statuses

| Status | Description |
|--------|-------------|
| `CREATED` | Enrollment session created |
| `READY_TO_ENROLL` | Customer has submitted details, ready to enroll |
| `ENROLLED` | Payment method successfully enrolled and vaulted |
| `ENROLL_FAILED` | Enrollment attempt failed |
| `EXPIRED` | Session expired before completion |
| `REJECTED` | Enrollment rejected by provider |
| `DECLINED` | Enrollment declined |
| `UNENROLLED` | Previously enrolled method has been removed |

## Payment result states

```swift
enum Result {
    case reject, success, fail, processing, internalError, userCancell
}

func yunoPaymentResult(_ result: Yuno.Result) { ... }
func yunoEnrollmentResult(_ result: Yuno.Result) { ... }
```

| State | Description |
|-------|-------------|
| `success` | Transaction completed successfully |
| `fail` | Transaction failed due to an error |
| `processing` | Transaction is being processed (async methods) |
| `reject` | Transaction was rejected |
| `internalError` | Unexpected server-side error |
| `userCancell` | User cancelled the payment flow |

## YunoTheme customization

Customize the checkout appearance to match your app's design.

```swift
let appearance = YunoAppearance(
    primaryColor: UIColor(hex: "#6200EE"),
    backgroundColor: .systemBackground,
    textColor: .label,
    errorColor: UIColor(hex: "#E53E3E"),
    borderColor: UIColor(hex: "#E0E0E0"),
    cornerRadius: 12.0,
    borderWidth: 1.0,
    fontFamily: "Avenir",
    buttonStyle: .rounded
)
```

### Color properties

| Property | Type | Description |
|----------|------|-------------|
| `primaryColor` | `UIColor` | Primary accent color for buttons, links, and active states |
| `backgroundColor` | `UIColor` | Background color of the checkout sheet |
| `textColor` | `UIColor` | Primary text color |
| `errorColor` | `UIColor` | Color for error messages and invalid field states |
| `borderColor` | `UIColor` | Border color for input fields |

### Typography and layout

| Property | Type | Description |
|----------|------|-------------|
| `fontFamily` | `String` | Font family name (e.g., `"Avenir"`, `"Helvetica Neue"`) |
| `cornerRadius` | `CGFloat` | Corner radius for cards, buttons, and input fields |
| `borderWidth` | `CGFloat` | Border width for input fields |
| `buttonStyle` | `ButtonStyle` | `.rounded` or `.rectangular` |

### Dark mode

The SDK supports iOS dark mode automatically using dynamic colors (e.g., `.systemBackground`, `.label`). Provide semantic colors for automatic adaptation:

```swift
let appearance = YunoAppearance(
    primaryColor: UIColor { traitCollection in
        traitCollection.userInterfaceStyle == .dark
            ? UIColor(hex: "#BB86FC")
            : UIColor(hex: "#6200EE")
    },
    backgroundColor: .systemBackground,
    textColor: .label
)
```

## SwiftUI integration

### Full Checkout view modifier

```swift
struct CheckoutView: View {
    let checkoutSession: String
    @State private var showCheckout = false
    @State private var paymentResult: Yuno.Result?

    var body: some View {
        VStack {
            Button("Pay Now") {
                showCheckout = true
            }
        }
        .yunoFullCheckout(
            isPresented: $showCheckout,
            checkoutSession: checkoutSession,
            countryCode: "CO",
            onResult: { result in
                paymentResult = result
            },
            onCancel: {
                showCheckout = false
            }
        )
    }
}
```

### Seamless Checkout in SwiftUI

```swift
struct SeamlessCheckoutView: View {
    let checkoutSession: String
    let paymentMethodType: String
    @State private var showCheckout = false

    var body: some View {
        VStack {
            Button("Pay with \(paymentMethodType)") {
                showCheckout = true
            }
        }
        .yunoSeamlessCheckout(
            isPresented: $showCheckout,
            checkoutSession: checkoutSession,
            paymentMethodType: paymentMethodType,
            countryCode: "CO",
            onCreatePayment: { token, tokenData in
                await createPaymentOnServer(token: token)
                Yuno.continuePayment()
            },
            onResult: { result in
                handleResult(result)
            }
        )
    }
}
```

## Error types

| Code | Description | Resolution |
|------|-------------|------------|
| `.networkError` | Network connectivity issue | Check device connectivity and retry |
| `.authenticationFailed` | Invalid public API key | Verify key in Dashboard > API Keys |
| `.sessionExpired` | Checkout session has expired | Create a new checkout session |
| `.invalidConfiguration` | SDK misconfiguration | Check initialization parameters |
| `.providerError` | Payment provider rejected the transaction | Review error details and payment method |
| `.cancelled` | User cancelled the operation | No action needed |
