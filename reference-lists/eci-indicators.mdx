## ECI (Electronic Commerce Indicator) Values

ECI values indicate the outcome of 3D Secure authentication for card transactions.

### Visa / Amex / Discover / JCB

| ECI | Description | Liability Shift |
|-----|-------------|-----------------|
| `05` | Fully authenticated (3DS) | Yes — to issuer |
| `06` | Attempted authentication (issuer/cardholder not enrolled) | Yes — to issuer |
| `07` | Non-3DS transaction or authentication failed | No — merchant liable |

### Mastercard

| ECI | Description | Liability Shift |
|-----|-------------|-----------------|
| `02` | Fully authenticated (3DS) | Yes — to issuer |
| `01` | Attempted authentication (issuer/cardholder not enrolled) | Yes — to issuer |
| `00` | Non-3DS transaction or authentication failed | No — merchant liable |

## 3DS Versions

| Version | Description |
|---------|-------------|
| `1.0` | 3DS 1.0 — Legacy, uses pop-up iframe for authentication |
| `2.1.0` | 3DS 2.1 — Frictionless flow support, risk-based authentication |
| `2.2.0` | 3DS 2.2 — Enhanced data elements, SCA compliance |

<Note>
ECI values are returned in the payment response under the `three_d_secure` object when 3DS authentication is performed.
</Note>
