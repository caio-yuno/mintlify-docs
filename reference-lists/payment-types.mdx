## Payment Method Types

Use these values in the `payment_method.type` field when creating payments.

| Type | Description | Countries | Flow |
|------|-------------|-----------|------|
| `CARD` | Credit and debit card payments | All | Direct / 3DS |
| `PIX` | Brazilian instant payment system | BR | QR Code |
| `BOLETO` | Brazilian bank slip (boleto bancario) | BR | Voucher |
| `PSE` | Colombian bank transfer (Pagos Seguros en Linea) | CO | Redirect |
| `SPEI` | Mexican interbank transfer (SPEI) | MX | Redirect |
| `OXXO` | Mexican cash payment at OXXO stores | MX | Voucher |
| `EFECTY` | Colombian cash payment | CO | Voucher |
| `BNPL` | Buy Now Pay Later | Varies | Redirect |
| `GOOGLE_PAY` | Google Pay wallet | Varies | Token |
| `APPLE_PAY` | Apple Pay wallet | Varies | Token |
| `BANK_TRANSFER` | Direct bank transfer | Varies | Redirect |
| `CASH` | Cash-based payments at retail locations | Varies | Voucher |
| `WALLET` | Digital wallet payments | Varies | Token / Redirect |

## Payment Flow Types

| Flow | Description |
|------|-------------|
| **Direct** | Payment is processed immediately with card data |
| **3DS** | Requires 3D Secure authentication before processing |
| **QR Code** | Customer scans a QR code to complete payment |
| **Voucher** | A payment voucher is generated for the customer to pay |
| **Redirect** | Customer is redirected to a third-party page to complete payment |
| **Token** | Payment is processed using a tokenized payment method |
